Tiktoken 可视化工具使用示例

示例 1: 基础分词

这个示例展示了如何使用cl100k_base编码器对一段英文文本进行分词：

  1. 选择 "cl100k_base" 编码器
  2. 输入文本: "Hello world! This is an example of tokenization."
  3. 点击 "Encode" 按钮
  4. 观察分词结果

预期结果: 文本将被分解为多个词元 (tokens)，例如 ["Hello", "world", "!", ...]

示例 2: 使用特殊词元

这个示例展示如何处理特殊词元，如GPT模型使用的<|endoftext|>标记：

  1. 启用 "Allow Special Tokens" 选项
  2. 添加特殊词元: "<|endoftext|>"
  3. 在文本框中输入: "This is a test.<|endoftext|>Another message."
  4. 点击 "Encode" 按钮

预期结果: <|endoftext|>会被识别为单个特殊词元，而不是多个字符

示例 3: 可视化词元分布

这个示例展示如何查看词元的3D可视化：

  1. 输入一段较长文本（至少20个词元）
  2. 对文本进行编码
  3. 点击 "Visualize" 按钮
  4. 切换到 "3D Vectors" 标签

3D可视化将展示词元在空间中的分布，类似的词元会聚集在一起

示例 4: 编码和解码测试

通过编码再解码验证词元化的可逆性：

  1. 选择任意编码器
  2. 输入一段文本
  3. 点击 "Encode" 按钮
  4. 查看生成的词元
  5. 点击 "Decode" 按钮
  6. 比较原始文本和解码文本

如果一切正常，解码后的文本应与原始文本完全相同

示例 5: 不同编码器比较

这个示例比较不同编码器的分词差异：

  1. 输入文本: "Machine learning is fascinating!"
  2. 使用cl100k_base编码，记录词元数
  3. 切换到p50k_base，再次编码，比较词元数
  4. 切换到r50k_base，再次编码，比较词元数

不同编码器可能产生不同数量的词元，反映它们的训练方式差异
